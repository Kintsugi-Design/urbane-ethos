## Problem Statement

The Urbane Ethos prototype's current motion polish (commit `911aba1`) is well-built but generic. Button compress, card lift, focus halo, chip pulse — could appear on any nice site. To feel like *this* site specifically (warm, handcrafted, Kenya Hara reference, funnel for an early intervention centre), the motion needs to carry the same DNA as the visual language. Photography also needs to upgrade from blank `[REAL PHOTO REQUIRED]` boxes to "considered placeholders" with intent captions, because the site is ultimately a funnel tool for the physical business, not just a brochure.

## What Makes This Cool

- Hand-drawn paper-and-ink motion details (sage stamps, ink underlines drawing across, paper-corner unfurl chatbot panels) carry Kenya Hara's craft DNA into the interaction layer. Hard to fake. Says "this site, no other."
- Captioned photo placeholders communicate intent during client review without forcing launch-quality assets. Each caption voices what the final photo is meant to be — so the client reads the prototype as "this is what we plan" rather than "this is what we have."
- Whole-experience coherence: organic timing rebalance everywhere + 6 craft moments + section-anchor photography + YouTube embed scaffolding means the prototype reads as a real site, not a wireframe. Real photos and videos slot into the same captioned positions before launch with zero layout work.

## Constraints

- CSS-first. JS only when CSS cannot express the motion or behavior (state machines we already have: consent, chatbot, i18n; small new IntersectionObserver already exists).
- No locally-hosted multimedia. YouTube embeds for all video, lazy-load (thumbnail + play button → iframe loads on click) to keep initial page light.
- Free image sources only (Unsplash, Pexels, Met Museum CC0). Real photos via parental consent before launch.
- Reduced-motion users get the static end-state via existing `prefers-reduced-motion: reduce` gate.
- Audience sensitivity: parents of children with developmental concerns. No exploitative imagery, no patronizing motion, no stock-kid cheese.
- Prototype budget: ~3 days of human work split across two landings (Phase 1 motion, Phase 2 media).
- Must not regress axe-core (currently 0 serious/critical across all 8 pages).

## Premises (Phase 3)

1. **The site is a funnel tool for the physical business**, not just a brochure. Motion + photography + video must do conversion work, not decorate. Every craft moment should reduce friction or build trust.
2. **Paper-and-ink craft motion + organic timing rebalance** is the motion direction (D1). Hybrid: organic ease curves everywhere as a calm baseline, custom paper/ink motion on 6 specific moments as brand-distinctive punctuation.
3. **6 craft moments** is the right scope (D2): consent save, chatbot open, locale toggle, link hover, service card hover, personalization saved. More dilutes; fewer leaves gaps.
4. **Photography integrates as section anchors with considered placeholders + intent captions** (D3 extended). YouTube handles all video. Caption format on the visible page: `"<Subject description> · Placeholder via <Source>."` The launch-swap note ("Real photo with parental consent before launch") lives in `media._note` in common.json for client-facing communication, not in the visible caption.
5. **CSS-first, JS only when necessary.** Motion tokens (`--ease-paper`, `--dur-stamp`, `--scale-press`, etc.) extend the existing tokens.css. Real photos and real videos slot into the same captioned positions at launch with zero layout changes.

## Approaches Considered

### Approach A: Full hybrid in one sweep
Land everything (organic timing rebalance, 6 craft moments, 6-8 captioned photos, YouTube scaffolding) in a single branch. Single review milestone. Most efficient if confident. High blast radius; hard to roll back partial pieces if one direction misfires. ~3 days human / ~1.5 hours CC dispatched.

### Approach B: Motion-then-media (chosen)
Phase 1 lands motion polish: tokens + organic timing rebalance + 6 paper-ink craft moments. You review and calibrate. Phase 2 lands photography sourcing + caption layout + YouTube embed scaffolding. Two review milestones, each landing reviewable in isolation. Lower risk; matches the D1 "rate the motion" feedback pattern. ~3 days split / ~1 hour CC per phase.

### Approach C: Tokens-first then full hybrid
Extract motion as named tokens before building moments. Future motion changes become one-token edits. Slowest first landing; some tokens may need renaming after real use proves names. ~3.5 days / ~1.5 hours CC.

### Approach D: Minimum viable
Organic timing rebalance + 3 craft moments only + 3 anchor photos. YouTube scaffolding deferred. Ships in ~half day at the cost of half the scope.

## Recommended Approach

**B (Motion-then-media).** Two clear review milestones. Motion polish lands first, calibrates against your taste before media work commits. If motion direction misfires, media work hasn't locked in. Trades wall-clock for review safety. Aligns with the D1 feedback pattern ("you rated current polish 'okay'") — get the motion right before adding media-scope.

2. **Organic timing rebalance applied via tokens, not literals**:

3. **6 craft moments implemented**:

   - **Consent save** (delete the `consent-saved-in` keyframe entirely): the existing text + generic checkmark from `911aba1` is replaced by an SVG sage circle stamp prepended to the existing "Saved." text. The stamp draws via `stroke-dasharray` animation (circumference 82 offset → 0 over 360ms via inline `<animate>` element); a companion checkmark `<path>` starts drawing at 320ms (intentionally 40ms before the circle finishes, so the checkmark feels organic rather than mechanically gated) and completes over 160ms. The "Saved." text label remains in serif, wrapped in `<span aria-live="polite">` for screen-reader announcement; the SVG itself carries `aria-hidden="true"`. Stamp wrapper is `<span style="display:inline-block;transform:rotate(-2deg);margin-left:4px;margin-top:-2px">` — the slight offset and rotation are the handcraft cue. After `--dur-stamp + 200ms` (720ms total), the entire confirmation block fades to opacity 0 over 200ms, then `el.remove()` deletes it from DOM. **CSS-first violation justification**: the SVG is JS-injected because it must trigger from the existing consent state machine (`writeConsent()` callback), not from a CSS state change. Pure-CSS keyframes can't be triggered by JS state changes alone.

   - **Chatbot open** (delete the `panel-in` keyframe entirely): new keyframe `chatbot-unfurl` added to motion.css. `transform-origin: bottom right`, start state `scale(0.6) rotate(-1.5deg) translateY(8px)` → end state `scale(1) rotate(0) translateY(0)` with `--ease-paper` over `--dur-fold`. Opacity 0 → 1 in parallel. Rotation magnitude is intentionally small (-1.5deg) to avoid right-edge clipping on 360px viewports.

   - **Locale toggle**: new keyframe `page-turn-slide` added to motion.css. When `[aria-pressed]` flips between EN and BM buttons, the newly-active button receives `animation: page-turn-slide var(--dur-2) var(--ease-paper) both;` (translateX from 6px to 0, opacity from 0 to 1). The newly-inactive button gets a fade-out over `--dur-1` (no slide). Replaces the previous `chip-settle` reference on the locale-toggle buttons.

   - **Link hover** (new, CSS only):
     ```css
     a {
       background-image: linear-gradient(currentColor, currentColor);
       background-size: 0% 1px;
       background-position: 0 100%;
       background-repeat: no-repeat;
       transition: background-size var(--dur-draw) var(--ease-ink);
       text-decoration: none;
     }
     a:hover { background-size: 100% 1px; }
     ```
     On `:not(:hover)`, `background-size: 0% 1px` collapses from the right edge (deterministic via `background-position: 0 100%`). Multi-line links degrade gracefully — the underline draws across the bounding box only on the last line; acceptable for prototype. Excludes consent-banner and chatbot-panel inner links (they keep their button-like styling).

   - **Service card hover** (refines `a.card:hover`): paper-corner lift. Card markup gets an inner wrapper `<div class="card-inner">` so the rotation applies to the wrapper, not the `.card` itself — this prevents neighbor-card misalignment in the grid during hover. CSS:
     ```css
     a.card .card-inner {
       transition: transform var(--dur-2) var(--ease-paper),
                   box-shadow var(--dur-2) var(--ease-paper);
       will-change: transform;
     }
     a.card:hover .card-inner {
       transform: translateY(-2px) rotate(-0.4deg);
       box-shadow: -6px 8px 16px -8px rgba(43, 31, 20, 0.18);
     }
     ```
     Shadow offset toward top-left suggests the card being lifted at its top-left corner.

   - **Personalization saved**: extract the sage-stamp SVG to a shared module at `assets/js/sage-stamp.js` exporting `renderSageStamp(targetEl)`. Both `consent.js` and `personalization.js` import and use it. **Identical inject/remove timing as consent save** (320ms draw, 720ms total visible, 200ms fade, then `el.remove()`).
