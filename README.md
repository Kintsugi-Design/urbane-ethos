# Urbane Ethos — Website Revamp Prototype

A Phase 2 interactive HTML prototype for the urbaneethos.center revamp.

Real scraped content (EN), draft Bahasa Malaysia translations, mocked-but-interactive chatbot + personalization, PDPA consent banner + privacy notice, WCAG 2.2 AA accessibility, and three design-direction comparison demos (B is the committed direction; A & C are artifacts).

- Live site (current): https://www.urbaneethos.center/
- Spec: [`docs/superpowers/specs/2026-06-08-urbane-ethos-revamp-design.md`](docs/superpowers/specs/2026-06-08-urbane-ethos-revamp-design.md)
- Plan: [`docs/superpowers/plans/2026-06-08-urbane-ethos-revamp.md`](docs/superpowers/plans/2026-06-08-urbane-ethos-revamp.md)
- A11y notes: [`docs/A11Y_NOTES.md`](docs/A11Y_NOTES.md)

## Status

**Phase 2 prototype. Not production.** EN content is scraped verbatim from the live site; BM is machine-translated and needs human + legal review before launch (especially `privacy.html`). Drafted English copy filling live-site gaps is flagged with a root `_draft` map in each JSON file.

## Run locally

```
bundle install
bin/server         # http://localhost:8080
```

Requires Ruby ≥ 3.1. No Node required at runtime. The a11y audit uses `npx @axe-core/cli` (no install — Node only needed if you want to re-run that audit).

## Design principles realized

1. **Personalization** — home micro-survey reorders the services grid and surfaces a "Recommended for you" rail. Client-side rules, gated by consent.
2. **Conversational design** — floating chatbot with scripted decision tree (Book / Service / Pricing / Human), voice input via Web Speech API + TTS via SpeechSynthesis, bilingual.
3. **More personal** — real human photographs front and center; staff page shows 9 team members with personal-line greetings, bios, and a "video coming soon" placeholder. Every missing image flagged `[REAL PHOTO REQUIRED]` in `alt`.
4. **Accessibility** — WCAG 2.2 AA with AAA contrast where cheap. Font-size toggle, skip-link, `prefers-reduced-motion`-gated animations, semantic landmarks, focus-visible everywhere, axe-core clean.

## What's real vs draft vs mocked

- **Real:** all EN copy that exists on the live site, scraped verbatim. Service blurbs, staff names + credentials + scraped bios, address, hours, phone numbers, blog posts (4 posts).
- **Drafted (flagged `_draft: true` in JSON):** copy filling live-site gaps — hero subtitle, values band, staff personal lines, drafted bios for 5 of 9 staff, per-service `whatItIs`/`whoItsFor`/`whatToExpect` where missing, FAQs, events teaser, prototype privacy notice, prototype consent strings.
- **Draft BM translations:** machine-generated by Claude with a glossary applied first. `_meta.reviewedBy: null` in every MS file. Glossary at `content/glossary.md`. Needs Malaysian native-speaker review before launch.
- **Mocked:** chatbot replies (scripted decision tree, no LLM), personalization rules (no ML — hard-coded concern→service mapping in `personalization.js`), staff video thumbnails (placeholder `<dialog>` modal), analytics data (seeded fake data in `analytics-demo-data.js`), contact form submission (`mailto:` only — no backend).
- **Placeholder images:** every missing photo carries `aria-label="[REAL PHOTO REQUIRED] <name>"` so reviewers can grep for the marker.

## Browser support

Modern only — Safari 16+, Chrome 110+, Firefox 110+. The site uses CSS cascade layers, modern custom properties, `clamp()`, `:where()`, native `<dialog>`. No transpile, no polyfills.

## i18n parity check

```
bin/check-i18n-parity.rb
```

Exit 0 = all keys mirrored between `content/en/` and `content/ms/`. `chatbot.json` is in scope. `blog.json` is intentionally EN-only.

## Known a11y gaps

Axe-core 4.11 reports **0 violations on all 8 production pages** (`/`, about, staff, services, blog, contact, analytics, privacy). Residual items not covered by the automated sweep:

- **Design-direction comparison pages** (`design/directions/v1-quiet/`, `v2-warm/`, `v3-bold/`) are internal artifacts, not linked from production routing, and intentionally excluded from the AA target.
- **Chatbot panel** is built lazily on launcher click, so axe-core only sees the static snapshot. Static structure was hand-inspected (labelled buttons, `role="log"` + `aria-live="polite"` on the message log); a follow-up `axe-core/playwright` pass that opens the panel first is recommended.
- **Google Maps iframe** on `/contact.html` carries a parent-level `title`; the iframe's *inner* DOM is third-party and outside this repo's a11y surface.
- **Manual / VoiceOver checks not yet automated:** chatbot focus trap + `Escape` close, contact form error announcement after submit, skip-link landing on `#main-content`, arrow-key navigation across personalization chips, and `prefers-reduced-motion` short-circuiting in `motion.css`.

Full details in [`docs/A11Y_NOTES.md`](docs/A11Y_NOTES.md).
